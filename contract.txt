01. Overview
This document defines the expected JSON response contract for the GET /api/properties endpoint of the
Prophoria (SmartRent) platform. It serves as the authoritative reference for both backend and frontend engineers,
ensuring consistent data structure, field naming, and type expectations across all client integrations.

02. Database Mapping Notes
The JSON response for this endpoint is an aggregated projection drawn from five relational database tables. The
backend join logic composes each property object by querying and merging data as described below:

Note on Arabic Content: The title and description fields may contain Arabic Unicode text. Client applications must ensure
proper RTL rendering and UTF-8 handling for these fields.
isFavorite field: This field is computed server-side based on the authenticated user's session. Unauthenticated requests will
always receive false. Backend should resolve this via a LEFT JOIN on the UserFavorites table filtered by userId.
